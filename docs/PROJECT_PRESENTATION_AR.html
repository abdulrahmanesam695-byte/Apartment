Project Explanation for Doctor

شرح مشروع CasaVista بالعربي

هذا المستند مكتوب بطريقة بسيطة جداً ليساعدني على فهم المشروع وشرحه بسهولة، خصوصاً لأنني ما زلت في بداية تعلم تطوير الويب.

Front-End Only HTML CSS JavaScript Local Storage

1. فكرة المشروع

المشروع عبارة عن موقع عقارات خاص ببيع وتأجير المنازل. المستخدم يستطيع مشاهدة العقارات، البحث بينها، فتح التفاصيل، حفظ المفضلة، وإرسال طلب معاينة. كما أن صاحب الموقع يستطيع متابعة كل شيء من خلال لوحة تحكم Dashboard تعرض التحليلات والإيرادات وحالة العقارات.

2. ما الهدف من المشروع؟

هدف للمستخدم

  • تصفح العقارات بسهولة
  • البحث والتصفية حسب الحاجة
  • رؤية التفاصيل قبل اتخاذ القرار

هدف للإدارة

  • متابعة عدد العقارات
  • معرفة المبيعات والإيجارات
  • متابعة الرسائل والإيرادات

3. لماذا المشروع Front-End فقط؟

لأن المطلوب في هذه المرحلة هو بناء الواجهة الأمامية فقط. لذلك لم أستخدم Back-End أو قاعدة بيانات حقيقية، واعتمدت على Local Storage لكي أحفظ البيانات داخل المتصفح نفسه.

ما الذي يتم حفظه؟

  • المفضلة
  • طلبات المعاينة
  • الأنشطة الأخيرة

الفائدة

  • الموقع يصبح تفاعلي
  • يمكن تجربة النظام بدون سيرفر
  • سهل للفهم لمشروع دراسي

الحدود

  • البيانات محلية فقط
  • لا توجد قاعدة بيانات حقيقية
  • لا توجد مشاركة بين الأجهزة

4. الأقسام الرئيسية داخل الموقع

Overview

يعرض مقدمة سريعة عن الموقع مع إحصائيات مختصرة وأزرار للتنقل.

Listings

يعرض العقارات في شكل بطاقات مع صور وخلفيات ومعلومات سريعة.

Details

يعرض تفاصيل العقار مثل الوصف، المميزات، المساحة، والغرف.

Dashboard

يعرض التحليلات، توزيع الحالات، الإيرادات، والرسائل.

5. كيف يعمل المشروع؟

1. عند فتح الموقع يتم تحميل بيانات العقارات الجاهزة.
2. إذا كانت هناك بيانات محفوظة مسبقاً في Local Storage يتم استخدامها.
3. المستخدم يستطيع البحث والتصفية بين العقارات.
4. عند الضغط على عقار يتم عرض التفاصيل الكاملة له.
5. المستخدم يمكنه حفظ العقار في المفضلة أو إرسال طلب معاينة.
6. الإدارة يمكنها تغيير حالة العقار من لوحة التحكم.
7. بعد أي تغيير يتم تحديث الإحصائيات والإيرادات مباشرة.

6. حالات العقارات والتحليلات

حالات العقارات

  • For Sale = معروض للبيع
  • For Rent = معروض للإيجار
  • Sold = تم بيعه
  • Rented = تم تأجيره

ما الذي يظهر في الـ Dashboard؟

  • عدد العقارات الكلي
  • عدد العقارات للبيع والإيجار
  • عدد العقارات المباعة والمؤجرة
  • إجمالي الإيرادات
  • الرسائل والأنشطة الأخيرة

7. كيف تم حساب الإيرادات؟

عند البيع

إذا كانت حالة العقار Sold يتم احتساب السعر الكامل كإيراد بيع.

عند الإيجار

إذا كانت حالة العقار Rented يتم احتساب الإيجار الشهري × 12 كإيراد سنوي.

8. شرح ملفات المشروع

index.html

يحتوي على الهيكل الأساسي للواجهة وأقسام الصفحة.

styles.css

مسؤول عن التصميم والألوان والتخطيط والاستجابة للموبايل.

app.js

مسؤول عن البيانات، التفاعل، الفلاتر، التفاصيل، والـ Dashboard.

9. نقاط القوة في المشروع

  • مشروع واقعي يجمع بين واجهة مستخدم ولوحة تحكم.
  • يوجد صور للعقارات وليس مجرد بيانات نصية.
  • فيه بحث، تصفية، تفاصيل، مفضلة، ورسائل.
  • فيه Dashboard تعرض التحليلات بشكل واضح.
  • مناسب جداً لمستوى طالب مبتدئ لأنه يوضح فهم الأساسيات بشكل عملي.

10. ما الذي يمكن تطويره لاحقاً؟

  • إضافة Back-End وربطه بقاعدة بيانات.
  • إضافة تسجيل دخول للمستخدم والإدارة.
  • إضافة رفع عقارات جديدة من لوحة التحكم.
  • إضافة خريطة للموقع الجغرافي.
  • إضافة أكثر من صورة لكل عقار.
  • إضافة إرسال بريد إلكتروني أو إشعار عند الحجز.

11. كيف أشرح المشروع للدكتور؟

هذا المشروع عبارة عن موقع عقارات Front-End فقط لبيع وتأجير المنازل. المستخدم يستطيع تصفح العقارات والبحث بينها وفتح تفاصيل كل عقار وحفظ المفضلة وإرسال طلب معاينة. كما أنني أضفت Dashboard للإدارة تعرض عدد العقارات والحالات المختلفة والإيرادات والرسائل. وبما أن المشروع Front-End فقط استخدمت Local Storage بدلاً من قاعدة بيانات حقيقية حتى أجعل المشروع تفاعلياً. الهدف من المشروع هو إظهار فهمي لتنظيم الواجهات، التعامل مع البيانات، وإظهار التحليلات بطريقة عملية.

12. أسئلة متوقعة من الدكتور

هل المشروع متصل بقاعدة بيانات؟

لا، المشروع Front-End فقط، وتم استخدام Local Storage لحفظ البيانات محلياً.

كيف تحفظ الرسائل أو المفضلة؟

يتم حفظها داخل المتصفح في Local Storage حتى تبقى موجودة بعد إعادة التحميل.

ما الفرق بين جزء المستخدم وجزء الإدارة؟

المستخدم يتصفح العقارات ويرسل الطلبات، والإدارة تتابع التحليلات وتغيّر حالة العقارات.

لماذا هذا المشروع مناسب كمشروع دراسي؟

لأنه يجمع بين التصميم والتفاعل والبيانات والتحليل في مشروع واحد واضح ومتكامل.
