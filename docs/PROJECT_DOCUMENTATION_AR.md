# توثيق مشروع CasaVista بالعربي

## 1. مقدمة بسيطة

هذا المشروع هو **واجهة أمامية فقط Front-End** لموقع عقارات خاص ببيع وتأجير المنازل.

فكرة المشروع أن المستخدم يستطيع:

- مشاهدة العقارات المعروضة
- البحث والتصفية بين العقارات
- فتح تفاصيل كل عقار
- حفظ العقارات المفضلة
- إرسال طلب معاينة

وفي نفس الوقت يوجد **Dashboard** لصاحب الموقع لعرض التحليلات مثل:

- عدد العقارات الكلي
- عدد العقارات المعروضة للبيع
- عدد العقارات المعروضة للإيجار
- عدد العقارات المباعة
- عدد العقارات المؤجرة
- الإيرادات
- الرسائل أو طلبات المعاينة

## 2. ما المشكلة التي يحلها المشروع؟

في الواقع، كثير من الناس يريدون البحث عن منزل بسهولة، وفي نفس الوقت صاحب الموقع يريد يعرف حالة المنصة:

- كم عدد العقارات الموجودة؟
- كم عقار تم بيعه؟
- كم عقار تم تأجيره؟
- ما قيمة الإيرادات؟
- هل يوجد مهتمون بالعقارات؟

هذا المشروع يجمع **واجهة للمستخدم** و **لوحة تحكم للإدارة** في نفس التطبيق.

## 3. هدف المشروع

الهدف من المشروع هو تصميم نموذج واقعي لموقع عقاري حديث يوضح فهمي لأساسيات:

- HTML
- CSS
- JavaScript
- Local Storage
- تنظيم الواجهة
- عرض البيانات وتحليلها

## 4. نوع المشروع

هذا المشروع هو:

- Front-End فقط
- لا يحتوي على Back-End
- لا يحتوي على قاعدة بيانات حقيقية
- يعتمد على **Local Storage** لحفظ البيانات محلياً في المتصفح

## 5. لماذا استخدمت Local Storage؟

لأن المشروع مطلوب منه واجهة فقط، فـ Local Storage تعتبر حل مناسب للمحاكاة.

مثلاً يتم حفظ:

- العقارات المفضلة
- طلبات المعاينة
- آخر العقارات التي تم فتحها
- تغييرات الحالات داخل الـ Dashboard

وهذا يجعل المشروع يبدو تفاعلياً حتى بدون سيرفر.

## 6. أقسام المشروع الرئيسية

### أ. صفحة البداية Overview

تعرض فكرة الموقع بشكل سريع، مع عنوان رئيسي وأزرار تنقل وإحصائيات مختصرة.

### ب. قسم العقارات Listings

هذا هو الجزء الذي يشاهده المستخدم. يمكن من خلاله:

- البحث باسم العقار أو المدينة أو النوع
- التصفية حسب الحالة
- التصفية حسب المدينة
- التصفية حسب عدد الغرف
- التصفية حسب السعر
- ترتيب العقارات

### ج. قسم التفاصيل Details

عند الضغط على أي عقار، تظهر تفاصيله مثل:

- اسم العقار
- العنوان
- المدينة
- المساحة
- عدد الغرف
- عدد الحمامات
- الحالة
- الوصف
- المميزات
- خط سير الإعلان

وأيضاً يوجد نموذج لإرسال طلب معاينة.

### د. لوحة التحكم Dashboard

هذا القسم خاص بالتحليل وعرض بيانات الإدارة. وهو يعرض:

- بطاقات إحصائية سريعة
- توزيع العقارات حسب الحالة
- توزيع العقارات حسب المدينة
- توزيع العقارات حسب النوع
- الإيرادات
- آخر الأنشطة
- صندوق الرسائل
- نموذج إداري لتغيير حالة العقار

## 7. حالات العقارات في المشروع

كل عقار يمكن أن يكون في واحدة من الحالات التالية:

- `for-sale` = معروض للبيع
- `for-rent` = معروض للإيجار
- `sold` = تم بيعه
- `rented` = تم تأجيره

وعند تغيير الحالة من لوحة التحكم، تتغير التحليلات تلقائياً.

## 8. كيف يتم حساب الإيرادات؟

- إذا كانت الحالة `sold` يتم اعتبار السعر الكامل إيراد بيع
- إذا كانت الحالة `rented` يتم احتساب الإيجار الشهري × 12 على أنه إيراد سنوي

وبالتالي Dashboard تعرض:

- إجمالي الإيرادات
- إيرادات البيع
- إيرادات الإيجار
- متوسط سعر العقارات المباعة
- متوسط الإيجار الشهري للعقارات المؤجرة

## 9. شرح الملفات داخل المشروع

### 1. `index.html`

هذا الملف يحتوي على هيكل الصفحة بالكامل، مثل:

- الهيدر
- أقسام الصفحة
- أماكن عرض الكروت
- مكان عرض التفاصيل
- مكان عرض الـ Dashboard

### 2. `styles.css`

هذا الملف مسؤول عن تصميم الواجهة مثل:

- الألوان
- الخطوط
- البطاقات
- الأزرار
- التخطيط العام
- الاستجابة للشاشات الصغيرة

### 3. `app.js`

هذا الملف مسؤول عن المنطق البرمجي، مثل:

- بيانات العقارات الافتراضية
- التصفية والبحث
- عرض العقارات
- عرض التفاصيل
- حفظ المفضلة
- حفظ الرسائل
- تحديث الـ Dashboard
- التعامل مع Local Storage

### 4. مجلد `img`

يحتوي على صور العقارات التي تظهر كخلفيات في البطاقات وصفحة التفاصيل.

## 10. كيف يعمل المشروع بشكل مختصر؟

1. عند فتح الموقع يتم تحميل بيانات العقارات الافتراضية.
2. إذا كان هناك بيانات محفوظة في Local Storage يتم استخدامها.
3. المستخدم يمكنه البحث والتصفية بين العقارات.
4. عند اختيار عقار تظهر تفاصيله.
5. المستخدم يمكنه حفظ العقار في المفضلة.
6. المستخدم يمكنه إرسال طلب معاينة.
7. الإدارة يمكنها تغيير حالة العقار من الـ Dashboard.
8. بعد أي تغيير يتم تحديث الإحصائيات والرسائل والأنشطة.

## 11. مميزات المشروع

- واجهة عصرية ومنظمة
- سهولة استخدام للمبتدئ
- وجود بيانات تجريبية جاهزة
- وجود صور للعقارات
- وجود Dashboard داخل نفس المشروع
- حفظ البيانات محلياً بدون سيرفر
- مناسب كمشروع تخرج أو Capstone لشرح أساسيات الويب

## 12. حدود المشروع الحالية

لأن المشروع Front-End فقط، توجد بعض الحدود:

- لا يوجد تسجيل دخول حقيقي
- لا يوجد API
- لا توجد قاعدة بيانات حقيقية
- البيانات محفوظة فقط على نفس المتصفح
- لا يمكن مشاركة البيانات بين أجهزة مختلفة

## 13. أفكار تطوير مستقبلية

في المستقبل يمكن تطوير المشروع بإضافة:

- Back-End باستخدام Node.js أو PHP أو Django
- قاعدة بيانات مثل MySQL أو MongoDB
- صفحة تسجيل دخول للإدارة والمستخدم
- رفع عقارات جديدة من لوحة تحكم حقيقية
- رفع صور متعددة لكل عقار
- خريطة للموقع
- حجز موعد فعلي
- إرسال الإشعارات عبر البريد الإلكتروني

## 14. كيف أشرح المشروع للدكتور؟

يمكنني شرح المشروع بهذه الطريقة:

> هذا المشروع عبارة عن واجهة أمامية لموقع عقارات خاص ببيع وتأجير المنازل.  
> المستخدم يستطيع تصفح العقارات والبحث بينها وفتح التفاصيل وحفظ المفضلة وإرسال طلب معاينة.  
> كما أضفت Dashboard للإدارة تعرض عدد العقارات والحالات المختلفة والإيرادات والرسائل.  
> بما أن المشروع Front-End فقط فقد استخدمت Local Storage لمحاكاة حفظ البيانات بدون Back-End.  
> الهدف من المشروع هو إظهار فهمي لتطوير واجهات الويب، وتنظيم البيانات، وإظهار التحليلات بشكل عملي.

## 15. أسئلة متوقعة وإجابات بسيطة

### س: هل المشروع متصل بقاعدة بيانات؟

ج: لا، المشروع Front-End فقط، لذلك استخدمت Local Storage كبديل محلي.

### س: كيف تحفظ الرسائل أو التغييرات؟

ج: يتم حفظها داخل Local Storage في المتصفح.

### س: ما الفرق بين المستخدم والإدارة في المشروع؟

ج: المستخدم يتصفح العقارات ويرسل طلبات، أما الإدارة فتتابع التحليلات وتغير حالة العقارات.

### س: كيف يتم تحديث الإيرادات؟

ج: عند تحويل العقار إلى `sold` أو `rented` يتم تحديث الإيرادات تلقائياً.

### س: لماذا هذا المشروع مناسب كمشروع دراسي؟

ج: لأنه يجمع بين تصميم واجهات، تفاعل المستخدم، إدارة البيانات، والتحليلات في مشروع واحد.

## 16. طريقة تشغيل المشروع

- افتح ملف `index.html` في المتصفح
- تصفح العقارات
- افتح أي عقار لرؤية التفاصيل
- جرّب إرسال طلب معاينة
- افتح الـ Dashboard
- جرّب تغيير حالة العقار وشاهد التحليلات تتغير

## 17. خلاصة

هذا المشروع يوضح أنني أستطيع بناء واجهة متكاملة لموقع عقارات تحتوي على:

- عرض بيانات
- تفاعل مستخدم
- حفظ محلي
- لوحة تحكم
- تحليل وإحصائيات

ورغم أنه ليس متصلاً بسيرفر حقيقي، إلا أنه يقدم نموذج عملي قريب من التطبيقات الحقيقية.
